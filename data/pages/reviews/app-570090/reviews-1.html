\r\n\t\t\t\t\tGreat tool (and almost only one available) for writing multiple choice story flow. It combines writing tools with design and content management tools, and I love it for this. Articy:Draft is well-arranged and its basics are easy to learn, so you can jump into writing early on. I would appreciate to be able to script even in dialogue and flow fragments but as far as I know, this might be solved with tokens (I am yet to find if the rules for tokens explained in Unity tutorial are applicable for Unreal as well).
Also, Articy:D is very visual what I find very useful for keeping order and track of the progress.

Import to Unreal is a bit clunky but now, with extensive tutorials (both video and written), the implementation can be set up quite easily.

Overall, I can fully recommend Articy:D to anyone with an intention to create their own game/visual novel/gamebook or even comics.\t\t\t\t\t

\r\n\t\t\t\t\tGreat tool for story writing. I've been using with Unity and it has a great Unity plugin and exporter which work pretty flawlessly. The great thing about the Unity plugin is that when you export your Articy draft file, it actually exports it as CODE to Unity. This means if you access Articy Global Variables directly in your C# code, and you rename one of those variables in Articy and re-export, you will immediately see a compiler error in Unity. This is super helpful as you don't have to run into hard-to-fix runtime bugs, instead you can see it immediately at compile time and fix it.
The Unity plugin is extremely well written and designed. I've had a few questions/issues I've contacted the Articy developer team about, and they responded within a day or two; super helpful.


There is so much to say about Articy, hard to do so in a single review, but I would say :
If you need a dialogue system that is portable between game engines, then Articy is definitely one of the top in the class.

Now, there are some drawbacks to Articy, specifically since we work in a Unity-specific environment it would be nice to have something more native to Unity (the Dialogue System Unity package fills that gap, and also supports importing Articy files).

Overall, would recommend to anyone writing a dialogue heavy game (such as a visual novel or interactive fiction type game).\t\t\t\t\t

\r\n\t\t\t\t\tWe've been using articy:draft 3 as a design, scripting, and game database tool for a turn-based RPG project. Overall, it's been a great tool and one I'd easily recommend to inexperienced developers. That said, there are a number of things to keep in mind when you're considering articy for serious game development, as it has some serious snags that may deter us from using it for future projects.

First off, articy's interface is simply great. It supports multiple simultaneous windows and tabs, many different views for conversation flows, and unique panel layouts for different types of objects. Basically, the developers had a strong vision for how the tool should be used and they mostly nailed it. Creating conversations is a breeze, and enabled our writer to dive in and start hammering out story. Connecting nodes can be done in a number of ways and works as easily as it could. Although creating dialogue is a simple affair in articy, consider that this workflow may not be best for conceptualizing conversations. The process of creating nodes and assigning portraits, as easy as it is, may get in the way of the creative writing process depending on the individual. Consider keeping to a document-based format for conceptualization and move into articy later.

The export options are excellent as well. While we experienced friction trying to work with the Unity importer plugin, the XML export option is fantastic and flexible to work with. We use articy in conjunction with Dialogue System for Unity, another excellent plugin, which supports the XML export perfectly. We've yet to encounter any show-stopping issues with this approach.

That isn't to say the tool is without its issues and quirks. For one, the app's undo/redo functionality seems to mysteriously disappear after a time, requiring a restart to regain them. This can lead to some very frustrating moments, so save often. Additionally, the multi-user licenses can be prohibitively expensive for smaller developers. The price difference between 1-3 stand-alone licenses and a multi-user one over several years is staggering. Since we only have a few users, we've instead opted to modify a single project file stored in Dropbox across three standalone licenses. This can work, but has pitfalls of its own. Partition files can become corrupted, which can be saved with version control, but work can still be lost. Using this method also means that only one user can access the project file at a time, so you'll need to coordinate closely. While I do not know the economics behind the multi-user environment, a cheaper entry point would be great for smaller developers.

Lastly, and perhaps most importantly, I'm not sure about the future of the tool. Since we've started using it, there have been no meaningful feature updates from the developers. The tool has seen plenty of maintenance releases and bug fixes, but that's it. Most recently, they introduced a free version of the app, but there's been no indication that they intend to actually further develop the tool or even introduce new features. It's unfortunate because the tool has such a great foundation and it's just stagnating. It's good for what it is, but not for what it could be.

I'd consider our team to be power users of this tool and as such we've run into some frustrating limitations in utilizing it for our personal uses. If we saw any intent from the developers to continue developing the tool, we'd surely make product suggestions to help ease those frustrations. Features like being able to prefab dialogue nodes and instantiate them, resize certain parts of dialogue nodes (such as menu text) or even Mac support would give me faith in this tool for the long run. Instead, we've acknowledged it as a moment frozen in time.

Thankfully, it's a great moment; one that will see us to release.\t\t\t\t\t

Articy Draft 3
Who and what is this for?
This version of articy: draft 3 is well suited for indie/hobbyist developers and can be used for world creation, as a game design document, and even as a project planning tool (and more).

Usability
It is easy to get going as the built in tutorials explain the basics quite convincingly. The more advanced functionality might take a while longer to master though. But overall good user experience.

Collaboration
It is possible to use this version even when collaborating with a couple of people. Although, I prefer the subscription based team bundles when working with others as they come with pretty good collaboration features.

Support & Service
My experience with Support is very good. They have responded really fast and been helpful, friendly, and providing a nice service level in general, which I much appreciate.

Overall
Very useful game development tool for individuals and small teams of 2-3 people.
