The documentation is quite good, I was even able to make some mod (plugin) to help my team with my workflow.

If you are an intermediate/experienced developer working on a project with a strong narrative, I fully recommend AD3.

I have a few reservations with the Unity plugin because of the way I want things to work, so I export JSON (or .docx if for readproof) and made a wrapper to import that exported JSON data on demand.

As for the tool itself, it's an amazing data management system. It's missing one tiny feature (list of struct-like properties) to be the best thing ever created, but it might get there too, maybe. Hopefully. We'll see.

10/10 would recommend.\t\t\t\t\t

\r\n\t\t\t\t\tArticy is extremely useful to keep stories, dialogues and content organized and presented in a very clean and pleasing format.

The plugins, however, could still use a lot of love. What once was sorted in a very easy-to-follow flowchart is now shown as only a list, at least in Unity. How to implement your Articy content into the game is also left unclear. Trying to figure it out on your own by looking at the sample project (Manic Manfred) could be an option... if only Norton didn't detect it as malware, which is... odd, to say the least.

For what it is, Articy is great for planning your content, and as such, I recommend it. However, it has potential to be much more... or rather, to be so in a much more intuitive way. I think a series of tutorial videos, like the one that already exists for the base software, could go a long way to familiarize users with how these plugins work.\t\t\t\t\t

\r\n\t\t\t\t\tIn short, I highly recommend this software to anyone who is serious about making a video game. That said, here is my explanation as to why I say that. making a video game is not easy. You have a lot of tasks that need to be thought out and features to develop. In steps Articy:Draft 3 a tool that allows you to create your virtual world in one piece of software for export into you game engine of choice.

Now if you are like me and bought the Articy:Draft 2, you may ask yourself why upgrade? My answer, it depends upon your technical specs you want to get out of this software. Do you want to use XML and Excel or Word? Well then there isn't too many reasons why you would want to upgrade. Otherwise, I would upgrade as the JSON exporter alone made the choice for me. As a game developer, I needed JSON in order to read the exported documents in my proprietary Game Engine. The other upgrades are nice ease of life features, but the ability o produce JSON documents and the ability to make plugins was my deciding factor and I'd bet many other people made the same conclusion.

Also, a side note, I highly recommend you look into the voice over plugin for anyone who is working on a budgeted game. If you plan to add in voice acting, this is a great way to make sure everything is nice and easy to use for the people you hire to do your voice overs.\t\t\t\t\t

Usually this clears up a few things (like text "missing" while typing in conditions, but are really there, because once you reload the project, it's written there, makes it super hard to see what's there though and edit if there are any issues)

This last one crashed on me sadly. Pretty sure this is some memory issue.

Another thing, the tutorials are basically useless. I won't say they are 100% useless, because I did pick up a lot of tidbits but, they skip over A LOT of stuff, sometimes creating them in between videos. Instead of making some huge example, they could have cut it down by half and done it in the same time frame. I mean, by the end of the series, the amount of views is less than 25% of when it starts out. So you know people just got as frustrated as I did and didn't bother with it, potentially refunding or never using it again.

It's really annoying, because like I said, there are things I like about it, from being able to simply write out my GDD in here and also the Locations stuff, where you can basically plan on Level Design.

There are a few things missing to make this #1 all-encompassing Game Design/Dev tool.

  • Official Unreal Engine 4 support and integration (there's a plugin, it's open source because they don't care about UE4 and don't want to support it but it's buggy as heck and improperly being updated. No branch per engine version.)
  • A way to "balance attributes" , kind of how you do it in Excel with curves (you can export and reimport, but balance sheet has to be in Excel)
  • Ability to choose which sections to export in a document (since "Include collapsed sections" seems to not work, it always exports everything no matter what
  • Ability to name pins, especially useful when you have multiple ones in Dialogues, and then you submerge, unless you remembered exactly, you'll be going back and forth to make sure you have the right pin connected from the inside to outside
  • I'm not sure a complete Project Management system is needed (like Trello/Hacknplan), but it could nice to have. Or something basic where you can maybe do high-level scheduling/burndown.
  • A more visual and easier way to do Skill/Tech Trees with Flow. (So all skills are on screen at the same time)

Some things to keep in mind. The app seems revolved around Dialogue and Branches. So if your game has none of this, I'm not sure how you'd benefit from it truthfully. There needs to be more variety for it to useful for more genres. If they can make another set of videos in a different genre, that would be gold.

I'm developing a game that's semi-based in procedural generation, so I'm not even sure if I'll be able to take advantage of it myself. Only time will tell.

TL;DR:

Why you may want to get it:
  • You use Unity
  • You have linear or branching dialogue in your game
  • You'd like to plan out missions with nice visual information

Why you might not want to get it:
  • You use Unreal
  • You have no dialogue in your game
  • Everything else you can think of

Another thing to note, this app goes on sale every Steam Sale at 30%.

I'll keep this updated as I play more around with it.\t\t\t\t\t
